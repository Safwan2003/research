## Task 0: build src/vlm/reasoning_llava.py

Create a `LlavaVLReasoner` class that mirrors QwenVLReasoner's public
interface exactly (same method names/signatures), so it's a drop-in
replacement wherever QwenVLReasoner is used:

  - `__init__(self, model_name="llava-hf/llava-1.5-7b-hf", device="cuda")`
  - `baseline_reasoning(image_path, question, report_text) -> str`
  - `context_aligned_reasoning(image_path, question, report_text, feature_card_json) -> dict`
  - `stepwise_agentic_reasoning(image_path, question, report_text, radiomics_json, feature_card_json) -> dict`
  - reuse the SAME prompt builders from src/vlm/prompts.py
    (build_baseline_prompt / build_context_aligned_prompt /
    build_stepwise_prompts) and the same `_safe_parse_json` JSON-extraction
    logic — don't duplicate that, import/reuse it.

Model loading differs from Qwen2-VL: use `transformers.LlavaForConditionalGeneration`
and `transformers.AutoProcessor` with `llava-hf/llava-1.5-7b-hf`. Follow the
repo convention of keeping `import torch` / `from transformers import ...`
local to `__init__`, not at module top-level. The chat-template and image-input
call pattern may differ slightly from Qwen2-VL's processor — check the actual
LlavaProcessor API in the installed transformers version (requirements.txt
pins transformers>=4.45.0) rather than assuming it's identical to Qwen2-VL's;
adapt `_generate()`'s internals to whatever that processor actually expects,
while keeping the public method signatures identical to QwenVLReasoner's.

Give it the same `if __name__ == "__main__":` self-test-explanation block as
reasoning.py (it can't run standalone without GPU either).

## Tasks 1-6 (same Known Gaps as Safwan's list, LLaVA side)

1. GradCAM classifier — this is shared infrastructure (torchxrayvision into
   src/features/xai_gradcam.py), not model-specific. Coordinate with Safwan
   so it isn't done twice; whoever gets there first does it, the other pulls
   the result.
2. Real OpenI data — also shared; same coordination note as above.
3. Run your new LlavaVLReasoner the same way Safwan runs QwenVLReasoner:
   baseline vs context-driven output on one real study, confirm the
   structured-JSON behavior actually shows up (shorter evidence, calibrated
   uncertainty, non-empty limitations/safety_note).
4. **Ablation study, BLIP text-embedding column (Table 1).** Note: Table 1
   compares "Qwen" vs "BLIP" text-embedding columns, NOT LLaVA — the paper's
   Figure 2 caption is explicit about this. Extract radiomics + XAI +
   BLIP-derived text embeddings (e.g. via a BLIP model's text
   encoder/projection over each report) for the same ~1,000 OpenI studies,
   then call `run_ablation()` in src/ablation/ablation_study.py. Target
   shape (paper's "BLIP" column): text-only ~0.925 AUC, radiomics-only
   ~0.564, XAI-only ~0.514, full combo ~0.930. Log with `mode="ablation"`,
   `config_snapshot={"model_family": "llava-1.5-7b", "text_embedding_source": "blip", ...}`.
5. **Hallucination + agentic reasoning (Tables 2-3-4, LLaVA-1.5-7B column of
   Table 4).** Use your `LlavaVLReasoner.stepwise_agentic_reasoning()` on
   the 50-study subset. Same evaluation functions Safwan uses:
   `hallucination.average_hallucination_rate()`,
   `responsible_ai_metrics.compute_agentic_reasoning_stats()` (once on step0
   outputs, once on step1, to compare deltas), and
   `compute_responsible_ai_indicators()` + `text_metrics.evaluate_batch()`
   for the LLaVA-1.5-7B row across variants A0-A5 (target from Table 4:
   A4_image_text is the best-performing variant, ROUGE-L ~0.471,
   BERTScore-F1 ~0.764). Log with the appropriate mode each time.
6. CheXpert cross-check — repeat your ablation (BLIP text embeddings) on
   CheXpert. Expect the same reversed pattern the paper reports (radiomics
   beats text there) — that's Section 5.4's own finding, not a bug.
