## CURRENT FOCUS: CheXpert (Known Gap #6) — do this first, before anything OpenI-specific

This is now automated. From the repo root:

    MODEL=qwen2-vl bash run_master.sh
    # or just: bash run_master.sh   (and pick [1] Qwen2-VL at the prompt)

That handles GPU check, environment setup, the CheXpert download (once
you've put your SAS URL in secrets/chexpert_sas_url.txt — see
antigravity-prompt.txt), the GradCAM classifier (torchxrayvision, wired into
run_chexpert_eval.py's feature extraction), the Table-1-style ablation, and
hallucination/agentic-reasoning/text-metrics evaluation — logging everything
to results/experiment_log.json. `load_chexpert_dataset()`'s old stubs
(empty report_text, null label) are already fixed.

Target numbers to sanity-check your run's ablation output against (Section
5.4): radiomics ~0.71 AUC should BEAT text ~0.50 here — the reverse of
OpenI's pattern. That's the paper's own reported finding, not a bug — don't
treat it as one if you see it. The paper doesn't give exact target numbers
for CheXpert's hallucination/Table 3/4-style metrics (those tables are
OpenI-only) — the point there is whether the same qualitative trend holds
(context alignment still reduces hallucination / stays calibrated), not
hitting specific values.

If you want a smaller/faster run first to sanity-check the pipeline before
committing to the full 1,000/50-study run:

    MODEL=qwen2-vl N_ABLATION=100 N_AGENTIC=10 bash run_master.sh

To run the 7B variant for the Table 4 Qwen2-VL-7B row instead of the default
2B, edit `VLM_MODEL_NAME` in config.py to `"Qwen/Qwen2-VL-7B-Instruct"`
before running (or pass it through if you add a flag for it — currently
config.py is the single source of truth for which Qwen checkpoint loads).

A single-study sanity check outside the master script is still available:
`python run_pipeline.py single --image <path> --report <path>` — use a
CheXpert image/report pair for this instead of OpenI's.
